Data preparation

The data structure adheres to the Darwin Core standart, which is utilised by numerous biodiversity data providers, such as GBIF.

A dataset consists of at least the following two tables (classes according to Darwin Core):

Event: Table for all surveys

Contains one row for each survey. This table is the main table (core class according to Darwin Core) and must include a column eventID with unique IDs. All other columns are optional and can be added as needed:

  • locationID : If the dataset includes a table for the survey locations, a locationID is essential in order to associate the events with the survey locations.
  • Additional columns, such as the date of collection, can be added as needed.

Occurence: Table for the recorded taxa

The documentation of an organism at a specific event serves as the basis for calculating the indicators. The event ID is essential for associating the recorded taxa with individual surveys. Z
