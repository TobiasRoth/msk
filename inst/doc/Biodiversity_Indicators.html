Introduction

The msk package provides R functions for calculating biodiversity indicators from species occurrence data. It was developed for analysing data from the Swiss Modular Stepwise Procedure (Modulstufenkonzept, MSK), but works with any dataset following the Darwin Core standard.

This vignette is structured in three parts:

  1. Data structure — how to organise your data for use with msk
  2. Indicator functions — how each function works, illustrated with macroinvertebrate data (mzb)
  3. Complete example — a full analysis of Swiss NAWA fish monitoring data (fisch), including a custom biomass metric
library(msk)
library(dplyr)
library(ggplot2)
library(patchwork)

Data structure

The msk package expects data as a list with two data frames (tibbles), following the Darwin Core standard.

event — one row per sampling event:

Column Description Required
eventID Unique identifier for the event yes
eventDate Date of the sampling event no
locationID Identifier for the sampling location no

occurrence — one row per taxon observation:

Column Description Required
eventID Link to the event table yes
taxonID Identifier for the recorded taxon yes
individualCount Number of individuals observed no

The two tables are linked by eventID. Additional columns (e.g. year, altitude, scientificName) can be added freely — they are preserved by all package functions.

Indicator functions

The msk package provides two types of indicator functions:

  • Event-level functions add a new column to the event table and return the modified data list. They are chainable with %>%.
  • Group-level functions compute indicators across sets of events and return a separate summary tibble.

Alpha diversity

Alpha diversity is the number of distinct taxa per sampling event. calc_alphadiv() adds an alphadiv column to the event table:

dat <- mzb %>%
  calc_alphadiv()

head(dat$event)
#> # A tibble: 6 × 6
#>   eventID  eventDate  locationID  year altitude alphadiv
#>   <chr>    <date>          <int> <int>    <dbl>    <int>
#> 1 100_2012 2012-03-21        100  2012      436       12
#> 2 100_2015 2015-02-25        100  2015      436       19
#> 3 100_2019 2019-02-25        100  2019      436       28
#> 4 100_2023 2023-03-01        100  2023      436       29
#> 5 101_2012 2012-03-21        101  2012      445       10
#> 6 101_2015 2015-02-25        101  2015      445       12

Abundance

calc_abundance() supports two methods, selected with the method argument:

  • "total" (default): sum of individualCount per event. Reflects overall abundance but is sensitive to dominant taxa.
  • "mean_per_taxon": mean of per-taxon counts per event. Less sensitive to dominant taxa; useful for tracking whether individual species become more or less abundant on average across taxa of different commonness.

Both methods can be applied in sequence:

dat <- dat %>%
  calc_abundance(method = "total") %>%
  calc_abundance(method = "mean_per_taxon")

head(dat$event)

The result columns are named abundance (total) and abundance_mean (mean per taxon).

Gamma diversity

Gamma diversity is the total number of distinct taxa across a group of events. Unlike alpha diversity, this is a group-level metric, so calc_gammadiv() returns a summary tibble. Use group_by_col to compute it separately per year:

calc_gammadiv(mzb, group_by_col = "year")
#> # A tibble: 4 × 2
#>    year gammadiv
#>   <int>    <int>
#> 1  2012       81
#> 2  2015       84
#> 3  2019       82
#> 4  2023       83

Beta diversity

Beta diversity measures how different communities are between events. calc_betadiv() computes pairwise Simpson dissimilarity:

\[\beta_{sim} = \frac{\min(b, c)}{a + \min(b, c)}\]

where a = shared taxa, b and c = taxa unique to each event. The index ranges from 0 (identical communities) to 1 (no shared taxa) and is independent of richness differences.

calc_betadiv(mzb, group_by_col = "year")
#> # A tibble: 4 × 4
#>    year betadiv_mean betadiv_q25 betadiv_q75
#>   <int>        <dbl>       <dbl>       <dbl>
#> 1  2012        0.329       0.222       0.429
#> 2  2015        0.293       0.2         0.379
#> 3  2019        0.308       0.219       0.389
#> 4  2023        0.295       0.207       0.370

The output contains the mean and the 25th/75th percentile of pairwise dissimilarities within each group.

Chaining and visualisation

Event-level functions return the data list unchanged (except for the new column), so they compose naturally. The following pipeline computes alpha diversity and abundance for mzb in a single chain:

campaigns <- c(2012, 2015, 2019, 2023)

mzb_ind <- mzb %>%
  calc_alphadiv() %>%
  calc_abundance(method = "total")

alpha_mzb <- mzb_ind$event %>%
  group_by(year) %>%
  summarise(
    mean = mean(alphadiv),
    lo   = mean - qt(0.975, n() - 1) * sd(alphadiv) / sqrt(n()),
    up   = mean + qt(0.975, n() - 1) * sd(alphadiv) / sqrt(n()),
    .groups = "drop"
  )

abund_mzb <- mzb_ind$event %>%
  group_by(year) %>%
  summarise(
    mean = mean(abundance),
    lo   = mean - qt(0.975, n() - 1) * sd(abundance) / sqrt(n()),
    up   = mean + qt(0.975, n() - 1) * sd(abundance) / sqrt(n()),
    .groups = "drop"
  )

beta_mzb <- calc_betadiv(mzb, group_by_col = "year")

Index normalisation

To express indicators relative to a reference point (base = 100), use calc_indicator_index(). This is useful for comparing trends across indicators or datasets measured on different scales.

Here we normalise gamma diversity relative to the 2012 baseline:

gamma_mzb <- calc_gammadiv(mzb, group_by_col = "year")

gamma_indexed <- calc_indicator_index(
  gamma_mzb,
  value_col = "gammadiv",
  group_col = "year",
  ref_value = 2012
)
gamma_indexed
#> # A tibble: 4 × 3
#>    year gammadiv index
#>   <int>    <int> <dbl>
#> 1  2012       81  100 
#> 2  2015       84  104.
#> 3  2019       82  101.
#> 4  2023       83  102.
ggplot(gamma_indexed, aes(x = year, y = index)) +
  geom_hline(yintercept = 100, linetype = 2) +
  geom_point(size = 2) +
  geom_line() +
  scale_x_continuous(breaks = campaigns) +
  labs(
    title = "Gamma-Diversität MZB (Index)",
    x     = "Zeitperiode",
    y     = "Index [2012 = 100]"
  )

Calculating indicators

We use the standard msk functions for alpha diversity and abundance, then add a custom biomass metric derived directly from the occurrence table. This illustrates how the Darwin Core structure makes it straightforward to extend the standard workflow with dataset-specific metrics:

fisch_ind <- fisch %>%
  calc_alphadiv() %>%
  calc_abundance(method = "total")

# Custom metric: total biomass per event (sum of totalWeight_g)
biomass_per_event <- fisch$occurrence %>%
  group_by(eventID) %>%
  summarise(biomass_g = sum(totalWeight_g, na.rm = TRUE), .groups = "drop")

fisch_ind$event <- fisch_ind$event %>%
  left_join(biomass_per_event, by = "eventID")
