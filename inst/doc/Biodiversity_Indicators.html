Calculating Biodiversity Indicators with msk

Introduction

The msk package provides a collection of R functions for calculating biodiversity indicators from species occurrence data. It was developed for analysing data from the Swiss Modular Stepwise Procedure (Modulstufenkonzept, MSK), a set of standardised methods for surveying and assessing the status of water bodies in Switzerland. However, the functions work with any dataset that follows the Darwin Core standard.

The package currently provides functions for:

  • Alpha diversity – species richness per sampling event
  • Total abundance – sum of individuals per sampling event
  • Gamma diversity – total species richness across events
  • Beta diversity – Simpson dissimilarity between events
  • Index normalisation – express indicator values relative to a reference
library(msk)
library(dplyr)
library(ggplot2)

Data structure

The msk package expects data as a list with two data frames (tibbles), following the Darwin Core standard:

event – one row per sampling event (Darwin Core Event class):

Column Description Required
eventID Unique identifier for the event yes
eventDate Date of the sampling event no
locationID Identifier for the sampling location no

occurrence – one row per taxon observation (Darwin Core Occurrence class):

Column Description Required
eventID Link to the event table yes
taxonID Identifier for the recorded taxon yes
individualCount Number of individuals observed no

The two tables are linked by eventID. Additional columns following Darwin Core terminology can be added freely (e.g. samplingProtocol, habitat, scientificName).

Calculating indicators

All indicator functions follow the same pattern: they take a data list dat and return it with the computed indicator added. This allows chaining multiple calculations.

Alpha diversity

Alpha diversity is the number of distinct taxa recorded at a single sampling event. Use calc_alphadiv() to compute it:

dat <- dia %>%
  calc_alphadiv()

head(dat$event)
#> # A tibble: 6 × 3
#>   eventID  eventDate  alphadiv
#>   <chr>    <date>        <int>
#> 1 100_2011 2011-03-23       30
#> 2 100_2012 2012-03-21       24
#> 3 100_2013 2013-03-19       23
#> 4 100_2014 2014-03-20       24
#> 5 100_2015 2015-02-25       29
#> 6 100_2019 2019-02-25       36

The new column alphadiv contains the species richness per event.

Total abundance

Total abundance is the sum of individualCount per event:

dat <- dat %>%
  calc_abundance()

head(dat$event)
#> # A tibble: 6 × 4
#>   eventID  eventDate  alphadiv abundance
#>   <chr>    <date>        <int>     <dbl>
#> 1 100_2011 2011-03-23       30       500
#> 2 100_2012 2012-03-21       24      1007
#> 3 100_2013 2013-03-19       23       501
#> 4 100_2014 2014-03-20       24       501
#> 5 100_2015 2015-02-25       29       500
#> 6 100_2019 2019-02-25       36       500

Both indicators can be computed in a single chain:

dat <- dia %>%
  calc_alphadiv() %>%
  calc_abundance()

Gamma diversity

Gamma diversity is the total number of distinct taxa across all events (or within groups). Unlike alpha diversity, this is a group-level metric, so calc_gammadiv() returns a separate summary table rather than adding a column to the event table.

# Overall gamma diversity
calc_gammadiv(dia)
#> # A tibble: 1 × 1
#>   gammadiv
#>      <int>
#> 1      377

To compute gamma diversity per year, first add a year column to the event table, then pass the grouping column name:

dat$event$year <- as.integer(format(dat$event$eventDate, "%Y"))

gamma_by_year <- calc_gammadiv(dat, group_by_col = "year")
gamma_by_year
#> # A tibble: 7 × 2
#>    year gammadiv

#> 1  2011      149
#> 2  2012      186
#> 3  2013      120
#> 4  2014      131
#> 5  2015      223
#> 6  2019      208

Beta diversity

Beta diversity measures how different communities are between sampling events. The calc_betadiv() function computes pairwise Simpson dissimilarity:

\[\beta_{sim} = \frac{\min(b, c)}{a + \min(b, c)}\]

where a is the number of shared taxa, b and c are the numbers of taxa unique to each event. The index ranges from 0 (identical) to 1 (completely different) and is independent of richness differences.

# Overall beta diversity
calc_betadiv(dia)
#> # A tibble: 1 × 3
#>   betadiv_mean betadiv_q25 betadiv_q75
#>          <dbl>       <dbl>       <dbl>
#> 1        0.496         0.4       0.588
# Beta diversity per year
beta_by_year <- calc_betadiv(dat, group_by_col = "year")
beta_by_year
#> # A tibble: 7 × 4

#> 2  2011        0.442       0.368       0.516

Visualising results

The indicator outputs are standard tibbles that work seamlessly with ggplot2.

Alpha diversity over time

alpha_summary <- dat$event %>%
  group_by(year) %>%
  summarise(
    mean = mean(alphadiv),
    lo = mean - qt(0.975, n() - 1) * sd(alphadiv) / sqrt(n()),
    up = mean + qt(0.975, n() - 1) * sd(alphadiv) / sqrt(n())
  )

ggplot(alpha_summary, aes(x = year, y = mean, ymin = lo, ymax = up)) +
  geom_point() +
  geom_errorbar(width = 0.2) +
  geom_smooth(method = "lm", se = FALSE, linetype = 2) +
  ylim(0, NA) +

Index normalisation

To compare indicators on a common scale, use calc_indicator_index() to normalise values relative to a reference year (base = 100):

gamma_indexed <- calc_indicator_index(
  gamma_by_year,
  value_col = "gammadiv",
  group_col = "year",
  ref_value = min(gamma_by_year$year)
)
gamma_indexed
#> # A tibble: 7 × 3
#>    year gammadiv index
#>   <int>    <int> <dbl>
#> 1  2011      149 100  
#> 2  2012      186 125. 
#> 3  2013      120  80.5
#> 4  2014      131  87.9
#> 5  2015      223 150. 
#> 6  2019      208 140. 
#> 7  2023      277 186.
ggplot(gamma_indexed, aes(x = year, y = index)) +
  geom_hline(yintercept = 100, linetype = 2) +
  geom_point() +
  geom_line() +
  labs(
    title = "Gamma Diversity Index",
    x = "",
    y = paste0("Index [", min(gamma_by_year$year), " = 100]")
  )

Extending the package

The msk package is designed to be extended with additional indicator functions. Planned additions include:

  • IBCH (Indice Biologique Global Harmonisé) – for macroinvertebrate assessment
  • SPEAR Index – pesticide sensitivity indicator
  • Shannon diversity / Evenness – abundance-weighted diversity measures
  • Trend analysis – temporal trend estimation with mixed models

New indicator functions should follow the established pattern:

  1. Take a dat list with event and occurrence tables
  2. Compute the indicator
  3. Return the modified dat list (for event-level indicators) or a summary tibble (for group-level indicators)

This ensures all functions are composable and work with the same data structure.
